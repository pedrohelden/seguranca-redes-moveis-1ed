Chapter 11 – Security Evasion in Mobile Networks (Firewall Bypass)

So far, you have seen how networks are protected. But there is an important reality:

👉 Firewalls are not perfect.

Experienced attackers do not try to break the network directly — they bypass the protections.

This chapter explains exactly how that happens.

11.1 The fundamental problem

Telecom firewalls (SS7, Diameter, 5G) are based on:

  • Rules (allow / block)
  • Trust lists
  • Message validation

👉 The problem:

If something looks legitimate, it is usually allowed.

And that is exactly what attackers exploit.

11.2 Main evasion techniques

1. Trust abuse

The attacker routes traffic through a trusted operator.

Flow:

Attacker → Trusted intermediate operator → Target network

👉 The firewall accepts it because the origin is trusted.

Impact:

  • Bypass of origin filtering
  • Attacks appear legitimate

2. Identity manipulation (spoofing)

Modification of parameters such as:

  • MSISDN
  • IMSI
  • Global Title

👉 Goal: appear as a legitimate network element.

Example:

Real origin: external network
Presented identity: internal network

3. Abuse of legitimate messages

Instead of using “malicious” messages, attackers misuse valid ones.

MessageLegitimate useMalicious use
ATILocation queryTracking
SRIRoutingInterception
ULRRegistrationFlood / DoS

👉 This makes detection very difficult.

4. Low & slow attacks

Instead of high volume, attackers use low frequency:

  • One request per minute
  • Different targets
  • Distributed pattern

👉 Result:

It bypasses volume-based detection systems.

5. Attack fragmentation

The attack is split into multiple steps:

Step 1: obtain IMSI
Step 2: obtain location
Step 3: intercept communication

Each step alone looks harmless.

👉 Together, they form a full attack.

6. Origin rotation

The attacker constantly changes:

  • IP address
  • Origin operator
  • Network identity

👉 This defeats blacklist mechanisms.

11.6 Full bypass example

Scenario: SMS interception

1. Obtain IMSI (SendIMSI)
2. Send SRI via trusted network
3. Redirect SMS
4. Maintain low frequency

Why it works?

  • Trusted origin
  • Legitimate messages
  • No suspicious volume

Result:

👉 Interception without immediate detection

11.7 Why firewalls fail

  • Over-trust in partners
  • Static rule-based logic
  • Lack of event correlation
  • Single-layer visibility

👉 Security based only on blocking is not enough.

11.8 How to defend against evasion

Defense against evasion requires higher maturity:

  • Event correlation (multi-step attacks)
  • Behavioral analysis
  • Strong origin validation
  • Zero Trust approach
  • Continuous monitoring

👉 The rule changes:

“It is not enough to block the attack — you must understand the behavior.”

11.9 Conclusion

Attackers constantly evolve. Evasion proves that.

  • Firewalls are necessary
  • But not sufficient

👉 True defense lies in visibility, intelligence, and correlation.
