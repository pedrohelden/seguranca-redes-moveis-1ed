Chapter 8 – Practical Defense and Security Architecture in Mobile Networks

After understanding how attacks work in practice, the most important question arises:

How do you actually defend a real mobile network?

This chapter provides a practical view focused on implementation, operation, and continuous defense — how a real security team (SOC/NOC) actually works.

8.1 The fundamental principle of defense

The modern security model in mobile networks is based on a simple concept:

Zero Trust: never trust, always verify.

  • Do not trust external networks
  • Do not automatically trust partners
  • Validate every message and every request

👉 This is the opposite of the original SS7 model.

8.2 Layered defense architecture

Effective protection happens across multiple layers:

Layer Protection
Edge (Interconnect) SS7 / Diameter Firewall
Core Network Message validation
Application (5G) API Security (OAuth, TLS)
Monitoring Detection and response

👉 If one layer fails, another still protects.

8.3 Signaling firewall (SS7 / Diameter)

This is the main defense mechanism.

Main functions:

  • Filter messages by type (MAP, CAP, Diameter)
  • Validate origin (GT, realm, IP)
  • Block illegitimate messages
  • Apply partner-based policies

Practical example:

  • Block AnyTimeInterrogation coming from external networks
  • Allow SendRoutingInfo only for trusted partners

8.4 Attack-specific protection

📍 Tracking (location)

  • Block external ATI
  • Validate roaming context
  • Allow only authorized GMLC

📡 Interception

  • Validate SendRoutingInfo
  • Check IMSI vs origin consistency
  • Block suspicious redirection

🔓 Data leakage

  • Block SendIMSI
  • Filter SendIdentification
  • Restrict HLR/HSS queries

💸 Fraud

  • Control CAP (CAMEL)
  • Validate charging requests
  • Monitor abnormal patterns

⚙️ DoS

  • Rate limiting
  • Flood detection
  • Automatic blacklisting

8.5 4G security (Diameter)

Diameter brought improvements, but still relies on trust.

Main risks:

  • Diameter Storm
  • CCR-based fraud
  • AIR/ULR data leakage

Controls:

  • Diameter Edge Agent (DEA)
  • Peer-based rate limiting
  • Session validation

8.6 5G security (SBA)

In 5G, security changes completely:

  • API-based (HTTP/2)
  • Service-based architecture (SBA)

Risks:

  • API exposure
  • Lack of strong authentication
  • API abuse attacks

Protections:

  • OAuth2
  • mTLS (mutual TLS)
  • API Gateway
  • Service mesh

8.7 Monitoring and detection (SOC/NOC)

Defense is not only about blocking — it is about observing.

Attack indicators:

Symptom Possible attack
Many ATI requests Tracking
Routing changes Interception
Signaling spikes DoS / Storm
Unusual requests Reconnaissance

8.8 Response playbook

When an attack happens:

  1. Detect (alert)
  2. Classify (attack type)
  3. Block source
  4. Apply firewall rule
  5. Monitor impact

👉 Response time is critical.

8.9 Security maturity

Level Description
Basic Static rules
Intermediate Active monitoring
Advanced Automation + analysis
Elite Real-time detection + AI

8.10 Simple explanation

Imagine the network as a building:

  • SS7 → open door
  • 4G → distracted doorman
  • 5G → digital system with password

But real security is:

  • Cameras
  • Access control
  • Continuous monitoring

👉 Having a door is not enough — you must watch it.

8.11 Conclusion

Defending mobile networks requires:

  • Secure architecture
  • Strict signaling control
  • Continuous monitoring
  • Fast response

The threat evolves — defense must evolve faster.
