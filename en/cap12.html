Chapter 12 – Tooling and Attack Engineering

After understanding how attacks work and how they bypass defenses, the most important question arises:

👉 How are these attacks actually built in practice?

This chapter shows the attacker’s perspective — not in terms of code, but in terms of architecture, process, and mindset.

12.1 The attacker mindset

An attacker does not start by attacking. They follow a logic:

  • Minimize risk
  • Maximize return
  • Appear legitimate
  • Avoid detection

👉 Core rule:

“If it looks like normal traffic, it gets through.”

12.2 Components of an attack infrastructure

An attack in mobile networks typically involves:

Component Function
Network access Entry into SS7 / Diameter / 5G
Signaling node Message generation (MAP / Diameter)
Control layer Attack orchestration
Distributed infrastructure Rotation and evasion

👉 An attack is not a single action — it is a system.

12.3 Building an attack (macro view)

Example: user tracking

1. Identify target MSISDN
2. Obtain IMSI (if needed)
3. Send location request
4. Repeat at controlled intervals

Key elements:

  • Frequency control
  • Source selection
  • Response validation

12.4 Message engineering

The core of the attack lies in signaling messages.

Exploited characteristics:

  • Optional fields
  • Inconsistent parameters
  • Undocumented behaviors

Conceptual example:

Message: location query
Fields:
- Subscriber identity
- Request origin
- Service type

👉 Small changes can completely alter network behavior.

12.5 Attack automation

Modern attacks are automated.

Function Description
Scanner Finds valid targets
Executor Sends attack messages
Correlator Analyzes responses
Scheduler Controls timing and volume

👉 This enables large-scale attacks.

12.6 Attacks at scale

An attacker can operate at different levels:

  • Individual: specific target
  • Batch: list of users
  • Massive: millions of subscribers

Defense challenge:

👉 Distinguish legitimate usage from automated attacks.

12.7 Distributed infrastructure

To avoid blocking, attacks use multiple origins:

  • Different networks
  • Different countries
  • Different identities

Result:

👉 Harder correlation and response.

12.8 Integration with evasion (Chapter 11)

Tooling and evasion work together.

Tool Evasion technique
Scheduler Low & slow
Origin rotation Blacklist bypass
Payload control Legitimate messages

👉 The attack is adaptive.

12.9 Full example (attack engineering)

Scenario: OTP interception

Phase 1:
- Collect MSISDN

Phase 2:
- Obtain IMSI

Phase 3:
- Execute routing attack

Phase 4:
- Intercept SMS

Phase 5:
- Maintain persistence with low volume

Key points:

  • Each phase alone looks legitimate
  • Distributed execution
  • Low operational visibility

12.10 Attacker limitations

Not everything is easy for attackers:

  • Dependency on network access
  • Risk of detection
  • Technical complexity
  • Infrastructure requirements

👉 These can also be leveraged for defense.

12.11 What defenders should learn from this

Understanding the attacker changes everything:

  • Defense becomes proactive
  • Shifts to strategic thinking
  • Enables behavior anticipation

👉 New mindset:

“If I were the attacker, how would I do it?”

12.12 Conclusion

Attack engineering shows that:

  • Attacks are planned
  • They are automated
  • They constantly evolve

👉 Defense requires the same level of sophistication.

Those who understand the attacker… defend better.
