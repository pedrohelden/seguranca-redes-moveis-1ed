Chapter 5 – Real-World Cases and Incidents in Mobile Networks

After understanding how attacks and mobile network architecture work, it is time to look at the real world.

This chapter shows how these attacks actually happen, with practical examples and clear explanations of the kill chain — the step-by-step path an attacker follows.

5.1 What is a kill chain in mobile networks?

A kill chain is the sequence of steps an attacker performs to achieve their goal.

  • 1. Target discovery
  • 2. Information gathering
  • 3. Network exploitation
  • 4. Attack execution
  • 5. Persistence or continued exploitation

👉 In mobile networks, all of this can happen in seconds.

5.2 Case 1 – Location tracking (SS7)

🎯 Objective

Determine a user’s location.

🧨 Methods used

  • AnyTimeInterrogation
  • SubscriberLocationReport

🔗 Kill Chain

StageDescription
1. EntryAttacker accesses SS7 network via operator or partner
2. IdentificationObtains victim’s MSISDN
3. QuerySends AnyTimeInterrogation request
4. ResponseReceives location (MSC/VLR)
5. MonitoringRepeats requests for continuous tracking

Impact: real-time user tracking.

5.3 Case 2 – SMS interception (bank OTP)

🎯 Objective

Intercept authentication SMS messages.

🧨 Methods

  • SendRoutingInfo
  • UpdateLocation

🔗 Kill Chain

StageDescription
1. ReconnaissanceObtains victim’s phone number
2. ManipulationSends fake UpdateLocation
3. RegistrationNetwork believes user is in another MSC
4. InterceptionSMS is redirected
5. ExploitationOTP used for fraud

5.4 Case 3 – IMSI leakage

🎯 Objective

Discover the subscriber’s identity in the network.

🧨 Methods

  • SendIMSI

🔗 Kill Chain

StageDescription
1. EntryAccess to SS7 network
2. QuerySend SendIMSI request
3. ResponseReceives victim IMSI

StageDescription

5.6 Case 5 – Diameter Storm (4G)

⚠️ What is it?

Diameter Storm is a situation where a massive volume of Diameter messages floods the network, causing severe congestion.

It is not always a deliberate attack — it can also result from misconfiguration.

🎯 Effects

  • Core network elements (MME, HSS) overload
  • Severe network latency
  • User authentication failures

🔗 Kill Chain

StageDescription
1. TriggerFailure or attack generates high request volume
2. AmplificationAutomatic retries increase traffic
3. SaturationHSS/MME become overloaded
4. CollapseUsers cannot register
5. Cascading failureOther network elements start failing

🧠 Why does this happen?

  • Automatic retry mechanisms
  • Strong dependency between nodes
  • Lack of rate limiting

Impact: large-scale 4G network outage.

5.9 Conclusion

Mobile network attacks are not theoretical — they have already happened and continue to happen.

  • SS7 → direct and silent attacks
  • 4G → signaling and scaling failures
  • 5G → API exposure risks

The difference between an incident and a secure network is the ability to detect and respond quickly.
