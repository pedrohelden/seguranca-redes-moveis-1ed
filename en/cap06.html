Chapter 6 – SS7 in Depth

Now that you understand how attacks work, we will open the “black box” of SS7.

This chapter shows how the protocol works internally, why it is vulnerable, and how real attacks happen at the signaling level.

6.1 What SS7 really is

SS7 (Signaling System No. 7) is the system responsible for controlling the mobile network — it does not carry voice or data directly, but rather the commands that make everything work.

Examples of what SS7 does:

  • Locate users
  • Route calls
  • Deliver SMS
  • Manage roaming

Analogy:

  • Data (voice/internet) = cars
  • SS7 = traffic control system deciding where they go

6.2 SS7 architecture

Element Function
MSC Switching center (calls)
HLR Subscriber database
VLR Temporary subscriber registry
STP Signaling router
GMSC Gateway to other networks

👉 The STP is the “router” of SS7 — it decides where each message goes.

6.3 Protocol stack

LayerProtocolFunction
ApplicationMAPOperations (location, SMS)
TransactionTCAPSession control
NetworkSCCPAddressing
TransportMTPReliable delivery

Simple summary:

  • MAP = what to do
  • TCAP = how to talk
  • SCCP = where to send

6.4 Real flow – SMS delivery

StepAction
1SMS arrives at SMSC
2SMSC sends SendRoutingInfo
3HLR responds with user’s MSC
4SMS is delivered

👉 This simple flow is exactly what attackers exploit.

6.5 The big problem: trust model

SS7 was designed with one assumption:

“If it is connected to the network, it is trusted.”

Today this is false.

  • Third-party operators
  • Global interconnections
  • Unauthorized access

👉 Result: any connected network can send dangerous commands.

6.6 Example technical message (simplified)

MAP: SendRoutingInfo
IMSI: 724XXXXXXXXXXXX
MSISDN: +5511999999999

👉 This type of message can reveal location or call routing paths.

6.7 Detailed attack – SMS interception

🎯 Objective

Steal SMS (e.g., banking code)

🔗 Technical flow

StepMessageDescription
1UpdateLocationAttacker registers fake location
2CancelLocationRemoves original registration
3InsertSubscriberDataUpdates data in fake VLR
4ForwardSMSMS is redirected to attacker

Result: victim continues using the phone normally, but SMS is diverted.

6.8 Detailed attack – tracking

🎯 Objective

Discover user location

🔗 Flow

StepMessageDescription
1AnyTimeInterrogationRequests location
2ProvideSubscriberInfoHLR returns data

Result: precise location (sometimes cell-level accuracy).

6.9 Detailed attack – IMSI discovery

🎯 Objective

Discover SIM internal identity

🔗 Flow

StepMessageDescription
1SendIMSIRequests IMSI
2ResponseHLR returns IMSI

Result: foundation for further attacks.

6.10 How operators defend themselves

ControlDescription
SS7 FirewallFilters suspicious messages
WhitelistAllows only trusted networks
Origin validationChecks if source is legitimate
MAP blockingBlocks dangerous commands

Main rule:

❌ Do not trust external networks
✅ Allow only what is necessary

6.11 Business impact

  • Financial fraud
  • Bank account theft
  • Privacy violations
  • Reputation damage

👉 A single attack can affect thousands of users.

6.12 Conclusion

SS7 remains one of the largest attack surfaces in mobile networks.

  • Powerful architecture
  • Outdated trust model
  • High global exposure

Understanding SS7 means understanding the foundation of modern telecom attacks.
