Chapter 2 – Mobile Network Security (SS7, 4G and 5G)

Now that you already understand how mobile networks work, it is time to explore one of the most important — and least visible — topics in this universe: security.

Mobile networks have evolved significantly in speed and capacity, but security has not always kept pace with this growth. This has opened the door to attacks that can affect users, operators, and even financial systems.

2.1 The fundamental problem

“All operators are trustworthy.”

👉 Result: external networks can send malicious commands.

2.2 Evolution of mobile network security

GenerationProtocolSecurity model
2G / 3GSS7Implicit trust
4GDiameterTrust between partners
5GAPIs (HTTP/2)Service-based architecture
  • SS7 is vulnerable by design
  • 4G still relies on trusted partners
  • 5G exposes functions via APIs

2.3 Risk assessment methodology

  • Severity: Low (+1), Medium (+2), High (+3), Critical (+4)
  • MSISDN usage: +2
  • Location impact: +3
  • Intercept impact: +3
  • External origin: +2

👉 The higher the score, the higher the priority.

2.6 Priorities

  • 🔴 Critical: Tracking and interception
  • 🟠 High: Fraud
  • 🟡 Medium: DoS

2.7 Simple explanation

  • SS7 = no security
  • 4G = partial trust
  • 5G = exposed APIs

2.8 Protection

  • SS7/Diameter firewalls
  • API protection
  • Monitoring

2.9 Conclusion

Security has evolved, but risks still remain.

  • SS7 → vulnerable by design
  • 4G → vulnerable due to trust
  • 5G → vulnerable due to APIs
