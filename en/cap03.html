Chapter 3 – How Attacks Actually Happen (Telecom Kill Chain)

In previous chapters, you learned how mobile networks work and what the main attacks are. Now we take a step further: understanding how these attacks are combined in practice.

In the real world, attackers do not use isolated techniques. They follow a logical sequence, exploiting vulnerabilities in a chain. This sequence is called the Kill Chain.

---

3.1 What is a Kill Chain?

A kill chain represents the stages of an attack, from reconnaissance to final impact.

StageObjective
ReconnaissanceIdentify the target (e.g. phone number)
EnumerationCollect data (IMSI, location)
ExploitationManipulate the network
ImpactInterception, fraud, or DoS

👉 Key idea: each stage enables the next one.

---

3.2 Anatomy of a real attack (SMS interception)

🎯 Goal: capture authentication code (OTP)

Stage 1 – Reconnaissance

The attacker starts with a simple piece of data:

  • MSISDN (phone number)

This can be obtained easily (leaks, public records, etc.).

---

Stage 2 – IMSI extraction

Network protocols allow querying the subscriber identity:

  • SendIMSI
  • SendIdentification

Result:

  • IMSI exposed
  • Unique SIM identity revealed

👉 The IMSI is more sensitive than the phone number — it identifies the user inside the network.

---

Stage 3 – Location discovery

Now the attacker needs to know where the user is in the network:

  • AnyTimeInterrogation
  • ProvideSubscriberInfo

Result:

  • Current MSC/VLR
  • Approximate region

👉 This allows proper targeting of the attack.

---

Stage 4 – Routing manipulation

This is where the main attack happens:

  • SendRoutingInfo
  • ProvideRoamingNumber

Result:

  • Calls/SMS redirected
  • Traffic passes through attacker-controlled entity
---

Stage 5 – Interception

With routing manipulated:

  • Bank SMS messages are captured
  • OTP codes are stolen
  • Accounts can be compromised

💥 Final impact: breakdown of two-factor authentication.

---

3.3 Simplified technical flow

Sequence visualization:

MSISDN → IMSI → Location → Routing → Interception

👉 Each step depends on the previous one. Without IMSI, no location. Without location, no redirection.

---

3.4 Another scenario: fraud

🎯 Goal: generate unauthorized charges

  1. Use of CAMEL InitiateCallAttempt
  2. Activation of services via ActivateSS
  3. Manipulation of billing logic

Result:

  • Unauthorized calls
  • Financial impact on the operator
---

3.5 Attacks in 4G (Diameter)

Even with evolution, the kill chain logic remains:

StageExample
EnumerationAuthentication Information Request (AIR)
ManipulationFake Update Location
ImpactSession interception

👉 The protocol changes, but the attack logic remains the same.

---

3.6 Attacks in 5G (SBA / APIs)

In 5G, attacks target APIs:

  1. Unauthorized API access
  2. Subscriber database queries
  3. Session manipulation

Impact:

  • Data leakage
  • Session control abuse
---

3.7 Universal attack pattern

PhaseObjective
IdentityDiscover IMSI
LocationFind user position
ControlManipulate routing
ExploitationIntercept or fraud

👉 This pattern repeats across 2G, 3G, 4G, and 5G.

---

3.8 Where operators can block attacks

🔴 Network entry

  • Signaling firewalls
  • Blocking external messages

🟠 Logical validation

  • Consistency checks (IMSI vs origin)
  • Routing validation

🟡 Monitoring

  • Attack pattern detection
  • Behavior analysis
---

3.9 Simple analogy

Think of the network as a building:

  • SS7: anyone can enter
  • 4G: trusted visitors only
  • 5G: modern digital access system

The attacker:

  1. enters
  2. observes
  3. discovers identity
  4. takes control
---

3.10 Most important insight

The real risk is not a single attack.

It is the combination of multiple steps.

👉 Effective security must break the entire chain, not just individual events.

---

3.11 Conclusion

Mobile networks are complex systems, but attacks follow a simple logic:

  • Discover
  • Locate
  • Control
  • Exploit

Understanding this sequence is essential for effective defense.
