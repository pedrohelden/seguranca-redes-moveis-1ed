Chapter 9 – Defense Architecture in Mobile Networks

After understanding how attacks work and how protocols operate, we reach the most critical point for any operator:

How do you design an architecture that can truly resist attacks?

This chapter shows how defense is built in practice — not as theory, but as real network architecture.

9.1 The trust problem

Historically, mobile networks were built on trust:

  • SS7 trusts any connected network
  • 4G trusts roaming partners
  • 5G trusts service-to-service authentication

👉 This model has failed.

Modern architecture must assume any source can be malicious.

9.2 Core principle: Zero Trust in Telecom

Zero Trust means:

  • Validate every message
  • Verify identity and context
  • Apply explicit policies

In practice:

  • Do not automatically trust SS7 GT
  • Do not trust Diameter peers
  • Do not trust internal 5G APIs

9.3 Layered defense architecture

Effective security is not built on a single component — it is distributed.

Layer Function Example control
Interconnect Entry control SS7/Diameter Firewall
Core Logical validation HLR/HSS/UDM checks
Application Access control OAuth / API Gateway
Monitoring Detection SOC / Analytics

👉 Real defense = multiple layers working together.

9.4 SS7 Firewall (2G/3G)

The SS7 Firewall is the first line of defense against legacy attacks.

Main functions:

  • Filter by message type (MAP/CAP)
  • Validate Global Title (GT)
  • Control by country or operator
  • Inspect parameters (IMSI, MSISDN)

Blocking examples:

Attack Firewall action
AnyTimeInterrogation Block external origin
SendIMSI Deny response
SendRoutingInfo Allow only trusted partners

Important limitation:

The SS7 Firewall depends on rules — and rules can be bypassed.

9.5 Diameter Firewall (4G)

In 4G, protection operates at the Diameter protocol level.

Functions:

  • Peer validation (realm, host)
  • Session control
  • Rate limiting
  • Storm detection

Examples:

  • Block excessive ULR
  • Validate CCR to prevent fraud
  • Detect signaling loops

Key point:

Diameter is more structured than SS7, but still relies on trust between networks.

9.6 5G protection (API Security)

In 5G, security shifts completely: now we are protecting APIs.

Main components:

  • API Gateway
  • Service Mesh
  • OAuth2 authentication
  • mTLS (mutual TLS)

Secure flow:

  1. Service requests token
  2. Token is validated
  3. Communication occurs via mTLS
  4. Request is audited

👉 This brings telecom closer to cloud and microservices.

9.7 Allow-list vs Block-list

There are two main control models:

Model Description Risk
Block-list Blocks known malicious behavior High (new attacks may pass)
Allow-list Allows only explicitly authorized behavior Low (safer approach)

Best practice:

👉 Use allow-list whenever possible.

9.8 Context validation (the key secret)

It is not enough to inspect a message — you must understand context.

Examples:

  • Does IMSI belong to the network?
  • Does the message origin make sense?
  • Is the user roaming?

👉 Many attacks succeed because they look “valid” individually.

9.9 Network segmentation

An essential practice:

  • Separate internal and external traffic
  • Isolate critical functions (HLR, HSS, UDM)
  • Control cross-domain access

👉 Reduces impact of intrusions.

9.10 Rate limiting and abuse protection

Fundamental against volumetric attacks:

  • Limit requests per second
  • Detect abnormal spikes
  • Apply automatic throttling

Example:

If a network sends thousands of ATIs per second → automatically block it

9.11 Integration with monitoring

Defense architecture does not work without visibility.

  • Centralized logs
  • Event correlation
  • Real-time alerts

👉 A firewall without monitoring is blind defense.

9.12 Simple explanation

Imagine the network as an airport:

  • SS7 Firewall → passport control
  • Diameter Firewall → boarding control
  • API Gateway → digital authentication
  • Monitoring → cameras and security staff

👉 Real security happens when everything works together.

9.13 Conclusion

A secure architecture does not rely on a single element, but on the combination of:

  • Strict filtering
  • Context validation
  • Access control
  • Continuous monitoring

The goal is not to stop all attacks — but to make exploitation difficult, expensive, and detectable.
