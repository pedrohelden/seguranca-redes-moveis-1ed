Chapter 10 – Detection and Monitoring in Mobile Networks

If Chapter 9 showed how to protect, this chapter shows something even more critical in practice:

👉 how to identify attacks happening in real time.

In modern mobile networks, security is not only about blocking — it is about continuous visibility.

This is the role of environments such as SOC (Security Operations Center) and NOC (Network Operations Center).

10.1 The challenge of detection in telecom

Unlike traditional IT systems, mobile networks have unique characteristics:

  • Massive signaling volume (millions of messages per second)
  • Specialized protocols (SS7, Diameter, HTTP/2)
  • Legitimate traffic can look malicious
  • Attacks are often silent (low & slow)

👉 Result:

Detecting an attack in telecom is like finding a needle in a haystack in real time.

10.2 Types of detection

Type Description Example
Rule-based Known signatures Block external ATI
Behavior-based Deviation from normal patterns Abnormal ULR volume
Context-based Combination of events Tracking + interception
Risk-based Dynamic scoring Origin + frequency + impact

👉 Modern systems use a combination of these approaches.

10.4 Real detection example (practical case)

Scenario: tracking attack

1. External origin sends ATI
2. Repeated queries for same MSISDN
3. Short time interval between messages

Correlation:

  • Untrusted origin
  • High frequency
  • Targeted subscriber

Conclusion:

👉 Active tracking attempt

10.5 Detection in 4G (Diameter)

In 4G, attacks are more subtle and often look like legitimate traffic.

Example: early Diameter Storm signs

SignalInterpretation
Increase in ULRPossible failure or attack
RetransmissionsNetwork timeouts
High HSS CPUSaturation

👉 If not handled quickly, it can escalate into a full network collapse.

In 5G, the focus shifts to APIs.

Common indicators

IndicatorDescription

👉 Observability tools are essential here.

10.7 Response playbooks

Detection is only half the job. The other half is fast response.

Example: Tracking playbook

1. Detect suspicious ATI
2. Validate origin
3. Block in firewall
4. Add to blacklist
5. Monitor recurrence

Example: Interception playbook

1. Detect anomalous SRI
2. Check routing changes
3. Block origin
4. Restore configuration
5. Audit impact

Example: DoS playbook

1. Detect signaling spike
2. Identify source
3. Apply rate limiting
4. Redirect traffic
5. Escalate incident

10.8 Continuous monitoring

A mature operator monitors:

  • SS7 / Diameter / 5G API traffic
  • Subscriber behavior
  • Roaming partner events
  • Network health (CPU, latency, errors)

👉 Security becomes continuous operation, not a one-time event.

10.9 Real-world challenges

  • High false positive rates
  • Lack of cross-network context
  • Distributed attacks
  • Massive data volume

👉 That’s why automation and correlation are essential.

10.10 Conclusion

Protection is important — but seeing the attack happening is what truly saves the operation.

  • No visibility → invisible attack
  • No correlation → useless alert
  • No response → inevitable damage

👉 In telecom, whoever detects first wins.
