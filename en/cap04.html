Chapter 4 – Defense Architecture in Mobile Networks

After understanding how attacks work (Chapter 3), we now answer the most important question:

How does a mobile operator actually defend itself?

This chapter presents the complete defense architecture — from the perimeter to the core — based on real industry practices.

---

4.1 The problem with traditional defense

Many networks still follow an old model:

  • They trust partner networks
  • They block only known attacks
  • They do not analyze behavior

👉 This fails because modern attacks are:

  • Sequential (kill chain based)
  • Disguised as legitimate traffic
  • Distributed across multiple networks
---

4.2 Modern principle: Zero Trust Telecom

The current security model is based on:

“Never trust, always verify.”

Old modelModern model
Implicit trustContinuous validation
Simple blockingContext-aware analysis
Perimeter focusLayered defense
---

4.3 Layered defense architecture

Effective protection happens across multiple layers:

[ External Network ] → [ Firewall ] → [ Core Network ] → [ Monitoring ]

Each layer has a specific role.

---

4.4 Layer 1 – Perimeter protection

Objective:

Block malicious messages before they enter the network.

Controls:

  • SS7 Firewall
  • Diameter Firewall
  • API Gateway (5G)

What is filtered:

  • Messages from unknown networks
  • Dangerous commands (e.g., SendRoutingInfo)
  • Out-of-pattern traffic

👉 First line of defense.

---

4.5 Layer 2 – Protocol validation

Objective:

Verify whether the message is technically valid.

Examples:

  • Does the IMSI belong to the source network?
  • Is the sender allowed to send this message type?
  • Does the flow follow the correct sequence?
ValidationExample
OriginValid GT/Realm
ConsistencyIMSI vs MSISDN
SequenceCorrect message order

👉 Blocks spoofing and logical manipulation.

---

4.6 Layer 3 – Attack-specific protection

Here we apply rules for known attacks:

AttackMitigation
AnyTimeInterrogationBlock external origin
SendRoutingInfoAllow only trusted partners
SendIMSIFull external blocking
ProvideRoamingNumberValidate IMSI vs origin

👉 Signature-based defense.

---

4.7 Layer 4 – Monitoring and detection

Objective:

Detect attacks that bypass previous layers.

What to monitor:

  • Traffic patterns
  • Suspicious sequences (kill chain behavior)
  • Unusual message volume

Example:

SendIMSI → AnyTimeInterrogation → SendRoutingInfo

👉 This indicates an ongoing attack.

---

4.8 Layer 5 – Response and containment

When an attack is detected:

  • Immediately block the source
  • Isolate network elements
  • Apply dynamic rules

👉 Response time is critical.

---

4.11 Mapping to the Kill Chain

An effective defense blocks each stage of the attack:

StageDefense
IdentitySendIMSI blocking
LocationATI filtering

👉 The goal is to break the chain.

---

4.12 Simple analogy

Think of the network as a modern building:

  • Reception: firewall
  • ID badge: authentication
  • Cameras: monitoring
  • Internal security: logical validation

If any layer fails, the attacker gets in.

---

4.13 Common operator mistakes

  • Trusting international partners
  • Focusing only on firewalls
  • Not monitoring behavior
  • Overly permissive rules
---

4.14 Conclusion

Mobile network security requires:

  • Layered defense
  • Continuous validation
  • Active monitoring

👉 It is not enough to block attacks — the kill chain must be broken.
