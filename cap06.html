Capítulo 6 – SS7 em Profundidade

Agora que você já entende como os ataques funcionam, vamos abrir a “caixa preta” do SS7.

Este capítulo mostra como o protocolo funciona por dentro, por que ele é vulnerável e como os ataques realmente acontecem no nível de sinalização.

6.1 O que é o SS7 de verdade?

O SS7 (Signaling System No. 7) é o sistema responsável por controlar a rede móvel — ele não transporta voz ou dados diretamente, mas sim os comandos que fazem tudo acontecer.

Exemplos do que o SS7 faz:

  • Localizar usuários
  • Encaminhar chamadas
  • Entregar SMS
  • Gerenciar roaming

Analogia:

  • Dados (voz/internet) = carros
  • SS7 = sistema de trânsito que decide para onde eles vão

6.2 Arquitetura do SS7

Elemento Função
MSC Central de comutação (chamadas)
HLR Banco de dados do assinante
VLR Registro temporário do usuário
STP Roteador de sinalização
GMSC Gateway para outras redes

👉 O STP é o “roteador” do SS7 — ele decide para onde cada mensagem vai.

6.3 Stack de protocolos

CamadaProtocoloFunção
AplicaçãoMAPOperações (localização, SMS)
TransaçãoTCAPControle de sessão
RedeSCCPEndereçamento
TransporteMTPEntrega confiável

Resumo simples:

  • MAP = o que fazer
  • TCAP = como conversar
  • SCCP = para onde enviar

6.4 Fluxo real – entrega de SMS

PassoAção
1SMS chega no SMSC
2SMSC envia SendRoutingInfo
3HLR responde com MSC do usuário
4SMS é encaminhado

👉 Esse fluxo simples é exatamente o que o atacante explora.

6.5 O grande problema: modelo de confiança

O SS7 foi criado com uma premissa:

“Se está conectado na rede, é confiável.”

Hoje isso é falso.

  • Operadoras terceiras
  • Interconexões globais
  • Acesso indevido

👉 Resultado: qualquer rede conectada pode enviar comandos perigosos.

6.6 Exemplo técnico de mensagem (simplificado)

MAP: SendRoutingInfo
IMSI: 724XXXXXXXXXXXX
MSISDN: +5511999999999

👉 Esse tipo de mensagem pode revelar localização ou rota de chamadas.

6.7 Ataque detalhado – interceptação de SMS

🎯 Objetivo

Roubar SMS (ex: código bancário)

🔗 Fluxo técnico

EtapaMensagemDescrição
1UpdateLocationAtacante registra falsa localização
2CancelLocationRemove registro original
3InsertSubscriberDataAtualiza dados no VLR falso
4ForwardSMSMS enviado para atacante

Resultado: vítima continua usando o celular normalmente, mas SMS é desviado.

6.8 Ataque detalhado – rastreamento

🎯 Objetivo

Descobrir localização do usuário

🔗 Fluxo

EtapaMensagemDescrição
1AnyTimeInterrogationConsulta localização
2ProvideSubscriberInfoHLR responde com dados

Resultado: localização precisa (às vezes nível de célula).

6.9 Ataque detalhado – descoberta de IMSI

🎯 Objetivo

Descobrir identidade interna do SIM

🔗 Fluxo

EtapaMensagemDescrição
1SendIMSISolicita IMSI
2RespostaHLR retorna IMSI

Resultado: base para outros ataques.

6.10 Como operadoras se defendem

ControleDescrição
Firewall SS7Filtra mensagens suspeitas
WhitelistPermite apenas redes confiáveis
Validação de origemConfere se origem é legítima
Bloqueio de MAP sensívelBloqueia comandos perigosos

Regra principal:

❌ Não confiar na rede externa
✅ Permitir apenas o necessário

6.11 Impacto no negócio

  • Fraude financeira
  • Roubo de contas bancárias
  • Violação de privacidade
  • Danos à reputação

👉 Um único ataque pode afetar milhares de usuários.

6.12 Conclusão

O SS7 continua sendo uma das maiores superfícies de ataque das redes móveis.

  • Arquitetura poderosa
  • Modelo de confiança ultrapassado
  • Alta exposição global

Entender o SS7 é entender a base dos ataques modernos em telecom.
